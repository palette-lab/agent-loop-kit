# Loop quality bar

A loop is accepted when it is:

- Practical: a real user would copy it into an agent today.
- Bounded: it has explicit stop conditions and cannot run forever.
- Verifiable: it names checks and evidence rather than asking for vibes.
- Safe: it escalates irreversible, privileged, privacy-sensitive, and policy-sensitive actions.
- Portable: it can be used across agent tools without vendor-specific assumptions unless declared.
- Original: it does not copy proprietary text from another prompt library.

Reject loops that:

- say only "improve until perfect";
- lack evidence requirements;
- allow production changes without approval gates;
- depend on private tools without providing a generic fallback;
- are just a normal checklist with no iteration or stopping logic.
